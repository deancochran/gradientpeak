# Plan: Continuous Fluid Periodization (MVP Architecture)

## Phase 1: The Heuristic Reference Generator

- **Objective:** Build the engine that draws the ideal, continuous Target CTL curve based on sports science rules.
- **Strategy:**
  - Create a new module to generate a `ReferenceTrajectory` (an array of daily target CTL values).
  - Architect the generator as a pure mathematical projection service that outputs a continuous array of smoothed daily TSS and CTL values, completely decoupled from the user's actual planned or completed calendar events.
  - Implement dynamic taper scaling (7-28 days) based on the maximum distance/duration extracted from `GoalTargetV2`.
  - Implement reverse curve generation: start at the goal date's target CTL and trace backward using the user's maximum safe weekly ramp rate.

## Phase 2: Multi-Goal Trajectory Merging

- **Objective:** Handle overlapping goals and multi-peak seasons on a single continuous curve.
- **Strategy:**
  - Update the input vector to accept an array of all profile goals.
  - Implement logic to draw localized "micro-tapers" for B/C priority events that interrupt the build phase of an A-priority event.
  - Ensure the `ReferenceTrajectory` smoothly connects multiple peaks without dropping back to a zero-base unnecessarily (respecting residual training effects).

## Phase 3: MPC Objective Function Refactoring

- **Objective:** Shift the existing MPC solver from a standalone optimizer to a trajectory tracker.
- **Strategy:**
  - Update `evaluateWeeklyTssCandidateObjectiveDetails` in `projectionCalculations.ts`.
  - Change the primary objective from "maximize fitness" to "minimize Mean Squared Error (MSE) between projected CTL and the `ReferenceTrajectory` CTL".
  - Ensure safety penalties (overload, high ATL, low readiness) remain heavily weighted to force the MPC to deviate from the reference line when the user is fatigued.

## Phase 4: Micro-cycle Instantiation (DUP)

- **Objective:** Translate the continuous daily TSS targets generated by the MPC into specific workout structures.
- **Strategy:**
  - Implement Daily Undulating Periodization (DUP) logic to select workout types (Long, Threshold, VO2max) that fulfill the daily TSS requirement.
  - Enforce Polarized Training constraints (e.g., 80% low intensity, 20% high intensity) across the generated micro-cycles.

## Phase 5: Validation & Testing

- **Objective:** Ensure the Heuristic + MPC architecture generates safe, fluid, and mathematically sound training plans.
- **Strategy:**
  - Write unit tests for the `ReferenceTrajectory` generator, specifically testing dynamic taper scaling and multi-goal merging.
  - Write integration tests verifying that the MPC successfully tracks the reference line under normal conditions, but safely deviates when simulated fatigue (ATL) is artificially spiked.
  - Verify execution speed remains within acceptable limits for real-time API responses.
