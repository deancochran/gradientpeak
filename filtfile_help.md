### **Research Report: The Optimal Open-Source Standard for Performance Analysis Data**

This report details the investigation into finding the best open-source, publicly known standard for recording, submitting, and parsing activity data for performance analysis. The key requirements include compatibility with major platforms like Garmin, Wahoo, and Strava, and the ability to serve as a robust alternative to custom compressed JSON payloads.

***

### **1. Introduction: The Need for a Universal Standard**

In the world of athletic performance analysis, data is king [[1]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/goverment-id/documentai_SampleDocuments_US_DRIVER_LICENSE_PROCESSOR_dl3.pdf?authuser=0)[[2]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/W2/documentai_LendingDocAI_W2Parser_2020_0411-w2-hz-hw.pdf?authuser=0) . Athletes and coaches rely on a wealth of metrics—such as Training Stress Score (TSS), Intensity Factor (IF), Functional Threshold Power (FTP), VO2 max, and threshold heart rate—to track progress, plan training, and gain a competitive edge [[1]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/goverment-id/documentai_SampleDocuments_US_DRIVER_LICENSE_PROCESSOR_dl3.pdf?authuser=0)[[2]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/W2/documentai_LendingDocAI_W2Parser_2020_0411-w2-hz-hw.pdf?authuser=0)[[3]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/AR%20Post-Build%20User%20Reqs.pdf?authuser=0)[[4]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQFt0RWIyXa3xTHrEVTX-JcbyWCu6w85Y5J3RvoAXKAgCy8-Z3puqjhLhMhRFsfmCYaZt6EL9E5fe2ayHNb7KJP6um0NHorCYK7cpE9lGq2BDKepiv9ZLru5zxU7gaJRsO8luz0J69eTEWoU0Cm18yD8XXFSvOSfbpTPePIB80XboFZprDbEtnSYbT5ZI4gDHWGes6yB70NLPFIu) . However, the methods for recording and transferring this data are often fragmented .

Many platforms resort to using custom, compressed JSON payloads . While flexible, this approach creates a siloed ecosystem where developers must write and maintain bespoke code for each data provider . This process is inefficient, requires significant server-side resources for decompression and parsing, and lacks the interoperability that a universal standard would provide [[1]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/goverment-id/documentai_SampleDocuments_US_DRIVER_LICENSE_PROCESSOR_dl3.pdf?authuser=0) .

The ideal solution is an open-source, publicly documented standard that is rich enough to contain complex performance metrics, efficient for transfer, and widely adopted by the key players in the fitness technology industry [[1]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/goverment-id/documentai_SampleDocuments_US_DRIVER_LICENSE_PROCESSOR_dl3.pdf?authuser=0)[[3]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/AR%20Post-Build%20User%20Reqs.pdf?authuser=0) . This report evaluates the most prominent standards available today: GPX, TCX, and FIT .

### **2. The Contenders: An Overview of Activity Data Formats**

Three file formats dominate the landscape of activity data exchange . Each was developed with different goals, and their capabilities vary significantly [[3]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/AR%20Post-Build%20User%20Reqs.pdf?authuser=0) .

#### **GPX (GPS Exchange Format)**

*   **What It Is:** GPX is a light-weight, open XML schema designed primarily for the exchange of GPS data [[3]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/AR%20Post-Build%20User%20Reqs.pdf?authuser=0)[[5]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQFqpr7C8W_AeDHSgszLyoW2G8MY8bkm1M9uKA6slTwXXvsGEz9nLkdXOp6J6tdNwr_jzDzQMvHD8LpePcdw_B-PSzWQw0RQzzXuolBV2p8h1EeXpVnUBCOzFNMgxHclnfAMZojo0VphCL2QjgV4XYaoyM26GdviglYfik65k5GFVKOXIEz05hNjwE1KNEfD) . Its core purpose is to describe waypoints, routes, and tracks .
*   **Openness:** GPX is a true open standard, and its specification is publicly available and has been stable for many years .
*   **Data Support:** By default, GPX stores location (latitude, longitude), elevation, and timestamps . While the format is extensible and can include additional data like heart rate and cadence through extensions, this is not part of the core standard . Support for these extensions can be inconsistent across platforms, and it generally lacks the structure for advanced performance metrics like power .
*   **Efficiency:** Being an XML-based format, GPX files are human-readable but are not as compact as binary formats, leading to larger file sizes [[3]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/AR%20Post-Build%20User%20Reqs.pdf?authuser=0) .

#### **TCX (Training Center XML)**

*   **What It Is:** Developed by Garmin, TCX is an XML-based data format created to expand upon the capabilities of GPX for fitness-specific applications [[3]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/AR%20Post-Build%20User%20Reqs.pdf?authuser=0)[[5]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQFqpr7C8W_AeDHSgszLyoW2G8MY8bkm1M9uKA6slTwXXvsGEz9nLkdXOp6J6tdNwr_jzDzQMvHD8LpePcdw_B-PSzWQw0RQzzXuolBV2p8h1EeXpVnUBCOzFNMgxHclnfAMZojo0VphCL2QjgV4XYaoyM26GdviglYfik65k5GFVKOXIEz05hNjwE1KNEfD) .
*   **Openness:** The XML schemas for TCX are publicly hosted by Garmin, allowing developers to build compatible applications .
*   **Data Support:** TCX was a significant step up from GPX for athletes . It natively includes fields for key performance data such as heart rate, cadence, and power . It also structures data into "Laps," which is crucial for interval training analysis . It can even be used for indoor activities that don't have a GPS component .
*   **Efficiency:** Like GPX, TCX is an XML format . This makes it human-readable and easy to edit with a text editor, but it results in significantly larger file sizes compared to binary alternatives [[3]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/AR%20Post-Build%20User%20Reqs.pdf?authuser=0) .

#### **FIT (Flexible and Interoperable Data Transfer)**

*   **What It Is:** The FIT protocol is a binary file format developed by Garmin and is now the de facto standard for recording and storing fitness, health, and GPS activity data from devices like GPS watches and bike computers [[3]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/AR%20Post-Build%20User%20Reqs.pdf?authuser=0)[[6]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQG_i7E5YYYC1PJMwY6Y6O_jSh1H1KSKZlBV9zffsettPL9GfUve20vS7vDPgh3eOUJ10Tk6euHQ6AmDVWDNebCOGxmCA_J4L1-rRSmx8mzZvjKNqsWhKgEU44WdhTAxZy2SCo_Fb0JTt6WsvKU=)[[7]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQGt60NmmzZ0Jrz5QhiGpbomLxRkCOHCPsVEtp8eZKrZeFcW4cmCON0GXX11N8teTQwerb_raKHrEZqaoFjkaPJ03_T6VoY8R3tRc1Ld_x_ibk4k3GeYfzjKbWstMRHTMj-HcELubCuSciGwrfM4hdMXwZ3m5Rg8idb8XLk2l3ARLXQDwgn4pvBlvxUP51K2UlAW1L41KA==) .
*   **Openness:** While developed by Garmin, the FIT protocol specification and a full Software Development Kit (SDK) are publicly available from THIS IS ANT [[3]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/AR%20Post-Build%20User%20Reqs.pdf?authuser=0)[[7]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQGt60NmmzZ0Jrz5QhiGpbomLxRkCOHCPsVEtp8eZKrZeFcW4cmCON0GXX11N8teTQwerb_raKHrEZqaoFjkaPJ03_T6VoY8R3tRc1Ld_x_ibk4k3GeYfzjKbWstMRHTMj-HcELubCuSciGwrfM4hdMXwZ3m5Rg8idb8XLk2l3ARLXQDwgn4pvBlvxUP51K2UlAW1L41KA==)[[8]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQGmUW7jx8cJp0q08GPYQv_9Xx8IefRpPq9nGAFLl10gjA0mENLG56w3VThPLXN4NbnYPbIFEJrNFNJi0mbcdRJmENb4G0CztUDOL1OFIEeUJOkJMLfvXml2tuLweeUtTQph-BfdfcGM0fMURjPGavBEhQlIH_Hr99bMAGu7deyREidq-IQ1xg==) . This allows any developer or manufacturer to implement the standard .
*   **Data Support:** FIT is by far the most comprehensive format . It is designed to be flexible and extensible, capable of storing a vast array of data points, including :
    *   High-fidelity GPS tracks .
    *   Sensor data (heart rate, cadence, power, temperature) .
    *   Lap and split summaries .
    *   Device information and settings [[3]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/AR%20Post-Build%20User%20Reqs.pdf?authuser=0) .
    *   User profile information (age, gender, weight) [[3]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/AR%20Post-Build%20User%20Reqs.pdf?authuser=0)[[9]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQEts8sN_NS0k8HyPd8_b-0oYgPtzGREl0acEDgbeCxS9oy47uSokJUia11dA8uN-jeOSqRzr73SSUwKOGOV-RZrHjjTsDvXXpimW8XkPd_gmOSu6tivlme4DkVQkFKff1_V9qnVHICxEvbVV-dp_ftwP1uFwsH5tOUpopeC7PWrq2IxVeo=)[[10]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQGPQK4K4RVbVCJqQClrQSCm0W5YpPcbLUVW99kDqGzG0NLxHCsoG06ragErzMjzY13cQBj8nlplpdOC8Ovt42f8LhxWRrZGdkQb5wMQx9LOn83_wnk9grH98weGSsAN8fX3nB7Dw9kwBuhSrmPIY0ZQryW8g2RXKKLOBlN05UTcFtpqEiIg57QXjQ==) .
    *   User-defined training zones, FTP, and Threshold Heart Rate [[3]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/AR%20Post-Build%20User%20Reqs.pdf?authuser=0)[[9]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQEts8sN_NS0k8HyPd8_b-0oYgPtzGREl0acEDgbeCxS9oy47uSokJUia11dA8uN-jeOSqRzr73SSUwKOGOV-RZrHjjTsDvXXpimW8XkPd_gmOSu6tivlme4DkVQkFKff1_V9qnVHICxEvbVV-dp_ftwP1uFwsH5tOUpopeC7PWrq2IxVeo=) .
    *   Advanced calculated metrics (including Garmin's estimated VO2 max) [[3]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/AR%20Post-Build%20User%20Reqs.pdf?authuser=0)[[11]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQGwO0wnShwmzZ5lRHiAN7wuronfgnCosC5UroblTS5Ld8pBdOvMR56ou6YTPe42be1k04SKV2MfmqFzo9cdG5bE_-YIzOe0v2kjrirTP5otvr6n9cRqEu5PYUhFL3jWLOY5BVaOOLGYFUu5lcjxpIg=)[[12]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQGUGjuyKdJ2JGctnv9cj-PIU45SR7MKtHa9xvtbVndihPfZh_3g2VHjlILFzTWEdo1ylw8hhtrCKZxT6nMqcDI0njycEzU60abVi60VCyyLg5ttbbUAQtKCGDKp4gO7DFxD3ANYZSEVwUW8rfo9VU_iKZ0XKGQN0EVMxs5ApaaipzgnhYqTDSY-N6hELPHqyPP4dTedrsVDEel_Cpl88yIE4yKI3vk=)[[13]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQG-WK4jVW7RDFtgaNXCVFSpVawPUdvJzuDYZ0FvBsD9HKgoViaLPp0gsBcQbdkn5FbACW119KbhwKFJmUTCeADbziK5f3C961XETS9ZNb3T-BaaSZuIyfQW4w==) .
*   **Efficiency:** As a binary format, FIT files are significantly more compact and efficient than their XML counterparts (GPX and TCX) . This makes them faster to transfer and less resource-intensive to process, directly addressing the inefficiency of large JSON payloads .

### **5. Extracting and Modeling Advanced Performance Metrics**

The FIT protocol is the only standard that provides robust, native support for the full suite of advanced performance metrics and the underlying data required for sophisticated modeling [[3]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/AR%20Post-Build%20User%20Reqs.pdf?authuser=0)[[17]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQHw1wEPkN4Cia7YJC2SVU_OMHHce5HviwyFNYJyIiGRYmOrszxcfjNGjy9_n_iqHJ7qJmEZSDiE-9-9GTtZgSzf2olAyh0h7pqvoqZXCatx8KZFH5R2xdd9pUpRUGVLGiq12FXFqrzCkGRfkkLNEnVdX-Y=) .

#### **5.1 Extracting User Profile and Summary Data from FIT Files**

Beyond the second-by-second time-series data, FIT files contain crucial non-time-series summary data that provides context for analysis . This data is stored in specific "message types" within the binary file [[9]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQEts8sN_NS0k8HyPd8_b-0oYgPtzGREl0acEDgbeCxS9oy47uSokJUia11dA8uN-jeOSqRzr73SSUwKOGOV-RZrHjjTsDvXXpimW8XkPd_gmOSu6tivlme4DkVQkFKff1_V9qnVHICxEvbVV-dp_ftwP1uFwsH5tOUpopeC7PWrq2IxVeo=) .

*   **FTP (Functional Threshold Power) & Threshold HR:** These values are essential for calculating metrics like TSS and IF [[3]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/AR%20Post-Build%20User%20Reqs.pdf?authuser=0) . #### **5.2 Methodology for Calculating NP, IF, and TSS**

While a FIT file can store a user's FTP, the key metrics of Normalized Power (NP), Intensity Factor (IF), and Training Stress Score (TSS) are calculated from the time-series power data recorded during the activity [[1]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/goverment-id/documentai_SampleDocuments_US_DRIVER_LICENSE_PROCESSOR_dl3.pdf?authuser=0)[[21]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/W2/W2_Interactive.pdf?authuser=0)[[5]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQFqpr7C8W_AeDHSgszLyoW2G8MY8bkm1M9uKA6slTwXXvsGEz9nLkdXOp6J6tdNwr_jzDzQMvHD8LpePcdw_B-PSzWQw0RQzzXuolBV2p8h1EeXpVnUBCOzFNMgxHclnfAMZojo0VphCL2QjgV4XYaoyM26GdviglYfik65k5GFVKOXIEz05hNjwE1KNEfD) . An accurate FTP, defined as the highest power output you can sustain for approximately one hour, is the cornerstone of these calculations [[22]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/xml/2259314_6036168_MIRequest_54466770.XML?authuser=0)[[23]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/paystubs/documentai_LendingDocAI_Pay%20Slip%20Parser_0001-paystub-walmart-hv8.pdf?authuser=0)[[3]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/AR%20Post-Build%20User%20Reqs.pdf?authuser=0)[[4]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQFt0RWIyXa3xTHrEVTX-JcbyWCu6w85Y5J3RvoAXKAgCy8-Z3puqjhLhMhRFsfmCYaZt6EL9E5fe2ayHNb7KJP6um0NHorCYK7cpE9lGq2BDKepiv9ZLru5zxU7gaJRsO8luz0J69eTEWoU0Cm18yD8XXFSvOSfbpTPePIB80XboFZprDbEtnSYbT5ZI4gDHWGes6yB70NLPFIu) .

1.  **Calculate Normalized Power (NP):** NP provides a more accurate measure of the physiological cost of a workout than average power by smoothing the data and emphasizing high-intensity efforts [[3]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/AR%20Post-Build%20User%20Reqs.pdf?authuser=0)[[24]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/lending-files/Combined_LDAI_Classifier-2.pdf?authuser=0)[[21]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/W2/W2_Interactive.pdf?authuser=0)[[25]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/W2/Sample-W2.pdf?authuser=0) . The process involves calculating a 30-second rolling average of power, raising each value to the fourth power, averaging the results, and taking the fourth root of that average [[24]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/lending-files/Combined_LDAI_Classifier-2.pdf?authuser=0)[[26]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/W2/documentai_LendingDocAI_W2Parser_2020_0442-w2-hz-hw.pdf?authuser=0)[[27]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/paystubs/documentai_LendingDocAI_Pay%20Slip%20Parser_0009-paystub-walmart-ar8.pdf?authuser=0)[[28]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/lending-files/lending_multi_document.pdf?authuser=0)[[29]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/bank-statements/08_LP1_Financial-Document-Samples_01.28.19.pdf?authuser=0)[[3]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/AR%20Post-Build%20User%20Reqs.pdf?authuser=0) .
2.  **Calculate Intensity Factor (IF):** IF quantifies the intensity of a workout relative to your personal FTP (`IF = NP / FTP`) [[1]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/goverment-id/documentai_SampleDocuments_US_DRIVER_LICENSE_PROCESSOR_dl3.pdf?authuser=0)[[30]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/paystubs/documentai_LendingDocAI_Pay%20Slip%20Parser_0005-paystub-walmart-hv8.pdf?authuser=0)[[16]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/W2/Form_W-2%2C_2016.pdf?authuser=0)[[31]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQH1yqoy7pNa16rBQ8yz4gV6wJyL38SflLL2vO8Nf5kq1dUPJnOLFj6qmJp7-EvvKZfrcbBQ8Cj9q1O7QBHmPwoC_JrxHklxkfP2uuJSMIyh_u2sRU8mWXsE9G17bUTt-lVkP7qqTg==)[[32]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQF3WHy2-vJGZuzxtUwdit6h2UCqdmybQCG0PKQR00wkz9P0BTfgYyaLkRvt8ZiZ6WL_a54AfPoWuRArE4KreachbXgROUTyZ_Y87Np2HDGKacauOR_x4X4T0D4m) . A one-hour ride at your FTP yields an IF of 1.0 [[1]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/goverment-id/documentai_SampleDocuments_US_DRIVER_LICENSE_PROCESSOR_dl3.pdf?authuser=0)[[32]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQF3WHy2-vJGZuzxtUwdit6h2UCqdmybQCG0PKQR00wkz9P0BTfgYyaLkRvt8ZiZ6WL_a54AfPoWuRArE4KreachbXgROUTyZ_Y87Np2HDGKacauOR_x4X4T0D4m)[[17]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQHw1wEPkN4Cia7YJC2SVU_OMHHce5HviwyFNYJyIiGRYmOrszxcfjNGjy9_n_iqHJ7qJmEZSDiE-9-9GTtZgSzf2olAyh0h7pqvoqZXCatx8KZFH5R2xdd9pUpRUGVLGiq12FXFqrzCkGRfkkLNEnVdX-Y=) .
3.  **Calculate Training Stress Score (TSS):** TSS measures the overall training load by combining intensity (IF) and duration [[33]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/paystubs/documentai_LendingDocAI_Pay%20Slip%20Parser_0007-paystub-walmart-ar8.pdf?authuser=0)[[34]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/David-files/VP-2023-tax-returns-SIGNED-REDACTED%20(1).pdf?authuser=0)[[35]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/W2/documentai_LendingDocAI_W2Parser_2020_0012-w2-ar-09-df.pdf?authuser=0)[[36]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQHSgYd-IGW_MWyhRFkiwCmQE8Lwf28v1K2wrQ9_zTuTsmQ2xxVmlRnFmUPqVgnV_0ZnNlcxFZAO7CM8hjO54kPRrPn3nCXOvZpbhgSucPbAQszzsSrtBHxnCIQTSoRkcRMEfgaKIu04s6KpMMZm9IBsQ5lbRZPa6HY3pMvhnF7CnloNbHxZsSZQN_Jz_06Xow==)[[5]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQFqpr7C8W_AeDHSgszLyoW2G8MY8bkm1M9uKA6slTwXXvsGEz9nLkdXOp6J6tdNwr_jzDzQMvHD8LpePcdw_B-PSzWQw0RQzzXuolBV2p8h1EeXpVnUBCOzFNMgxHclnfAMZojo0VphCL2QjgV4XYaoyM26GdviglYfik65k5GFVKOXIEz05hNjwE1KNEfD) . The formula is `TSS = ( (Duration in seconds × NP × IF) / (FTP × 3600) ) × 100` [[22]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/xml/2259314_6036168_MIRequest_54466770.XML?authuser=0)[[33]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/paystubs/documentai_LendingDocAI_Pay%20Slip%20Parser_0007-paystub-walmart-ar8.pdf?authuser=0)[[37]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/paystubs/documentai_LendingDocAI_Pay%20Slip%20Parser_0004-paystub-walmart-hv8.pdf?authuser=0)[[23]](https://storage.cloud.google.com/essent-agentspace-tes/Essent-test-files/paystubs/documentai_LendingDocAI_Pay%20Slip%20Parser_0001-paystub-walmart-hv8.pdf?authuser=0)[[38]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/Arkansas%20Rate%20Manual%20Effective%2002-01-2023.pdf?authuser=0)[[36]](https://vertexaisearch.cloud.google.com/grounding-api-redirect/AUZIYQHSgYd-IGW_MWyhRFkiwCmQE8Lwf28v1K2wrQ9_zTuTsmQ2xxVmlRnFmUPqVgnV_0ZnNlcxFZAO7CM8hjO54kPRrPn3nCXOvZpbhgSucPbAQszzsSrtBHxnCIQTSoRkcRMEfgaKIu04s6KpMMZm9IBsQ5lbRZPa6HY3pMvhnF7CnloNbHxZsSZQN_Jz_06Xow==) .

#### Practical Example: Parsing a FIT file with `fitdecode`
The following Python code demonstrates how to parse a `.fit` file, extract key time-series data, and organize it into a Pandas DataFrame [[3]](https://storage.cloud.google.com/essent-agentspace-test2/New%20folder/AR%20Post-Build%20User%20Reqs.pdf?authuser=0) . ```python
import fitdecode
import pandas as pd

def fit_to_dataframe(fit_file_path: str) -> pd.DataFrame:
    """
    Parses a .FIT file using fitdecode and returns a Pandas DataFrame
    containing key time-series data from 'record' messages.

    Args:
        fit_file_path: The path to the .FIT file.

    Returns:
        A Pandas DataFrame containing the time-series data.
    """
    data_points = []
    desired_fields = [
        'timestamp', 'heart_rate', 'power', 'cadence', 
        'position_lat', 'position_long', 'enhanced_speed', 'enhanced_altitude'
    ]

    with fitdecode.FitReader(fit_file_path) as fit:
        for frame in fit:
            if isinstance(frame, fitdecode.FitDataMessage) and frame.name == 'record':
                row = {field: frame.get_value(field) if frame.has_field(field) else None for field in desired_fields}
                
                # Convert GPS coordinates from semicircles to degrees
                if row.get('position_lat') is not None:
                    row['position_lat'] /= ((2**32) / 360)
                if row.get('position_long') is not None:
                    row['position_long'] /= ((2**32) / 360)
                    
                data_points.append(row)
    
    return pd.DataFrame(data_points)

### **9. Executive Summary**
